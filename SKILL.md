# 出海潜力客户评估系统 v7.0

帮助销售团队精准识别高质量潜在客户，判断客户所处阶段，挖掘交叉销售机会的企业出海评估工具。

## 核心设计原则

1. **双轨评估机制** — 目标公司+母公司独立评估，挖掘集团层面机会
2. **三维动态权重** — 根据"出海阶段 + 企业体量 + 行业属性"自动计算权重
3. **数据溯源透明** — 每个维度得分必须附带数据来源、关键指标、数据缺口
4. **微弱信号探测** — 主动识别高价值的动态预警信号
5. **置信度透明** — 每次评估附带数据完整度评分
6. **竞品FOMO触发** — 挖掘竞争对手动作作为销售突破口
7. **集团关联挖掘** — 识别母公司其他出海活跃业务板块

## 第一步：三维修正识别

### 维度1：出海阶段

| 阶段 | 定义 | 核心问题 |
|-----|------|---------|
| **阶段一：出海初期** | 刚开始考虑出海，无实质动作 | "该不该出海？从哪里开始？" |
| **阶段二：出海成长期** | 已有1-2个海外项目，运营<3年 | "如何优化现有海外业务？" |
| **阶段三：出海成熟期** | 海外业务体系化，运营>3年 | "如何加速扩张？" |

### 维度2：企业体量

| 体量 | 判定标准 | 核心关注 |
|-----|---------|---------|
| **大型集团** | 年营收>50亿，或母公司年营收>100亿 | 战略布局、风险控制 |
| **中型企业** | 年营收5-50亿 | 增长与成本平衡 |
| **小型创新企业** | 年营收<5亿 | 市场增长、技术突破 |

### 维度3：行业属性

| 行业 | 特性 | 权重侧重 |
|-----|------|---------|
| 制造业 | 供应链本地化要求高 | 供应链×1.3 |
| 科技/互联网 | 技术适配，品牌重要 | 技术×1.3，品牌×1.2 |
| 消费品 | 品牌、渠道、本地化 | 品牌×1.3 |
| 服务业 | 人员本地化、合规重要 | 本地化×1.3 |
| 金融 | 合规、风控严 | 合规×1.4 |

## 第二步：双轨评估机制

### 核心逻辑

```
┌─────────────────────────────────────────────────────┐
│                   双轨评估                          │
├─────────────────────┬───────────────────────────────┤
│    轨道A：目标公司    │      轨道B：母公司            │
│  • 独立评估公司本体   │   • 评估母公司出海战略         │
│  • 判断独立出海价值   │   • 识别其他出海业务板块       │
│  • 若信息不足，基于   │   • 评估资金支持力度           │
│   业务特性评估       │   • 识别交叉销售机会           │
└─────────────────────┴───────────────────────────────┘
```

### 评估要求

| 情况 | 处理方式 |
|-----|---------|
| 目标公司有明确母公司 | 必须双轨评估 |
| 目标公司无母公司/独立公司 | 仅做轨道A，在报告中注明 |
| 目标公司信息极度匮乏 | 侧重轨道B，以母公司信息为主 |

## 第三步：11个评估维度（含数据溯源要求）

### 数据溯源核心要求（必须执行）

每个维度的评估必须包含以下四要素：

| 要素 | 说明 | 示例 |
|-----|-----|-----|
| **数据来源** | 打分所依据的具体信息来源 | 公开财报、新闻报道、招聘平台、公司官网 |
| **关键指标** | 支持打分的具体数值或事实 | "海外营收占比15%"、"员工人数2000人" |
| **数据缺口** | 无法获取的关键信息 | "海外市场具体投资金额未披露" |
| **推断依据** | 基于有限信息的推断逻辑 | "因无公开反对，假定管理层支持出海" |
